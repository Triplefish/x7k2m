# 🚀 5分钟快速部署指南

跟着这个指南，5分钟内完成基金估值工具的部署！

## 📋 准备工作（5分钟）

### 步骤 1：注册维格表账号

1. 打开浏览器，访问：https://vika.cn
2. 点击右上角「注册」
3. 使用手机号或微信快速注册
4. 登录后进入工作台

### 步骤 2：创建数据表

1. 点击左上角「+ 新建」→「空白表格」
2. 给表格起个名字，比如「我的基金估值」
3. 删除默认的列，保持空白即可（脚本会自动创建列）

### 步骤 3：获取 API Token

1. 点击右上角你的头像
2. 选择「用户设置」
3. 左侧菜单选择「开发者配置」
4. 点击「创建 Token」
5. 给 Token 起个名字（如：基金追踪工具）
6. 点击创建，**复制并保存**这个 Token（很重要！只显示一次）

### 步骤 4：获取数据表 ID

1. 回到你刚创建的数据表
2. 看浏览器地址栏的 URL，例如：
   ```
   https://vika.cn/workbench/dstXXXXXXXXXX/viwYYYYYYYYYY
   ```
3. 复制 `dst` 后面到下一个 `/` 之前的部分（dstXXXXXXXXXX）
4. 这就是你的数据表 ID，**保存下来：Mu1n5UVusylzZfT**

## 🔧 GitHub 配置（3分钟）

### 步骤 5：Fork 本仓库

1. 如果你还没有 GitHub 账号，先去 https://github.com 注册一个
2. 登录后，访问本项目页面
3. 点击右上角的「Fork」按钮
4. 等待几秒钟，仓库就复制到你的账号下了

### 步骤 6：配置 Secrets

1. 在你 Fork 的仓库页面，点击顶部的「Settings」（设置）
2. 左侧菜单找到「Secrets and variables」→ 点击「Actions」
3. 点击右上角绿色按钮「New repository secret」

**添加第一个 Secret：**

- Name（名称）：`VIKA_API_TOKEN`
- Value（值）：粘贴你在步骤3复制的 API Token
- 点击「Add secret」

**添加第二个 Secret：**

- 再次点击「New repository secret」
- Name（名称）：`VIKA_DATASHEET_ID`
- Value（值）：粘贴你在步骤4复制的数据表 ID
- 点击「Add secret」

### 步骤 7：启用 GitHub Actions

1. 点击仓库顶部的「Actions」标签
2. 如果看到提示，点击「I understand my workflows, go ahead and enable them」
3. 左侧列表点击「基金估值追踪」
4. 点击「Enable workflow」按钮

## 🎉 测试运行（1分钟）

### 步骤 8：手动运行一次

1. 在 Actions 页面，确保选中了「基金估值追踪」workflow
2. 点击右侧的「Run workflow」下拉按钮
3. 再点击绿色的「Run workflow」按钮
4. 页面会刷新，稍等几秒会出现一个黄色的运行任务
5. 点击进去可以看到运行详情
6. 等待 1-2 分钟，状态变成绿色 ✅ 就表示成功了

### 步骤 9：查看结果

1. 回到维格表（https://vika.cn）
2. 打开你创建的数据表
3. 你应该能看到 7 条基金数据，包括：
   - 基金名称
   - 昨日净值
   - 估算涨跌
   - 估算净值
   - 更新时间
   - 等等

### 步骤 10：添加到微信（可选）

1. 在维格表页面，点击右上角「分享」按钮
2. 复制分享链接
3. 在微信里打开这个链接
4. 点击右上角「···」→「添加到我的小程序」
5. 以后在微信里就能快速打开查看了！

## 🔧 自定义基金列表

如果想修改追踪的基金：

1. 在你 Fork 的仓库中，点击 `fund_tracker.py` 文件
2. 点击右上角的铅笔图标（编辑）
3. 找到 `FUNDS = [...]` 这个列表
4. 修改、添加或删除基金信息
5. 点击底部「Commit changes」保存

**基金配置示例：**

```python
# ETF联接基金
{
    "name": "易方达黄金ETF联接C",
    "code": "002963",              # 基金代码
    "type": "etf_linked",          # 类型
    "etf_code": "159934",          # 对应的ETF代码
    "etf_name": "黄金ETF"          # ETF名称
}

# 主动型基金
{
    "name": "南方信息创新混合A",
    "code": "007490",
    "type": "active",
    "index_code": "399006",        # 参考指数代码（创业板指）
    "index_name": "创业板指"       # 参考指数名称
}

# 债券型基金
{
    "name": "博时转债增强债券A",
    "code": "050019",
    "type": "bond",
    "index_code": "000832",        # 中证转债指数
    "index_name": "中证转债"
}
```

## ❓ 遇到问题？

### Actions 运行失败？

1. 检查 Secrets 是否配置正确
2. 查看运行日志，最下方会显示错误信息
3. 确保维格表 API Token 有效

### 数据不更新？

1. 确认是否在交易日的 9:30-15:00 之间
2. 手动触发一次 workflow 测试
3. 检查 Actions 是否正常运行

### 看不到数据？

1. 确认数据表 ID 是否正确
2. 确认 API Token 是否有效
3. 手动运行一次 workflow，查看日志
